# Volty(볼티) — 전기기사 학습 앱

직장인 대상 전기기사 필기 학습 앱. 폰으로는 출퇴근 중 짧은 복습, 패드/웹으로는
집중 이론 학습을 하는 구조.

**🔗 라이브 데모: https://yunj8649.github.io/volty/** (폰·PC 브라우저에서 바로 실행)

기능: 이론 카드 198장 · 학습 진척/즐겨찾기/검색 · 단원 연습문제 · 모의고사(회차별·랜덤·직접설정, 볼티 원작 200문항) · 오답노트

## 왜 문제은행 앱이 아닌가

조사 결과 **전기기사 필기 기출문제의 합법적 원본 소스는 존재하지 않는다.**

- 공단은 필기 문제지를 공개하지 않는다. 응시자가 시험 후 문제지를 직접 가져가는
  방식이라 인터넷 공개를 하지 않는다는 게 공식 입장이다. Q-Net 자료실의
  `기출문제` 게시판은 **기술사 전용**이고, `공개문제` 게시판은 **실기**다.
- 2022년 3회차부터 필기가 CBT로 전환되어 그 이후 문제는 아예 비공개다.
  시중 CBT 사이트 데이터가 전부 2022년에서 끊기는 이유다.
- 시중 사이트의 2003~2022 문제는 PBT 시절 응시자가 들고 나온 문제지를 수집·복원한
  것이고, 저작권은 여전히 출제기관에 있다. comcbt는 자체 약관으로 상업적 이용을
  금지하고 있다.

그래서 문제 데이터로는 아무도 차별화할 수 없고(모두 같은 pool), 그 링에는
comcbt·기출넷·맞추다·다산패스가 이미 무료로 들어와 있다.

대신 **개념 커버리지**로 간다. CBT는 응시자마다 다른 문제가 나오고 공단 문제은행도
`PBT 기출 + 신규`라, 특정 문제 원문보다 개념 커버리지가 중요해졌다. 그 커버리지의
공식 정의가 출제기준이다.

## 구조

```
data/taxonomy/    공단 출제기준에서 뽑은 개념 분류 트리 (5과목 / 680개념)
tools/            PDF -> 트리 -> 앱 에셋 파이프라인
app/              Flutter 앱 (iOS / Android / Web 단일 코드베이스)
```

### 개념 분류 트리

`한국산업인력공단 전기기사 출제기준(필기)` (적용기간 2024.01.01~2026.12.31)에서
추출. 공단이 공시하는 문서라 저작권이 자유롭고, 출제기관이 직접 정의한 분류라
시중 어떤 학원 자료보다 정확하다.

`과목 > 주요항목 > 세부항목 > 세세항목` 4단계이고, 세세항목(개념) 680개가
학습의 최소 단위다. 학습 기록·약점 진단·이론 카드가 전부 개념 ID에 매달린다.

| 과목 | 주요 | 세부 | 개념 |
|---|---:|---:|---:|
| 전기자기학 | 8 | 55 | 195 |
| 전력공학 | 7 | 37 | 151 |
| 전기기기 | 7 | 57 | 164 |
| 회로이론 및 제어공학 | 2 | 19 | 92 |
| 전기설비기술기준 | 5 | 30 | 78 |
| **합계** | **29** | **198** | **680** |

## 이론 카드

**카드 1장 = 세부항목 1개, 총 198장.** 세세항목(680개)이 아니다 — 개별로는 두세
줄짜리 토막이라 따로 읽을 이유가 없고, 상당수가 "등"으로 끝나 공단 스스로 목록을
열어뒀다. 세세항목은 카드가 반드시 덮어야 할 **체크리스트**(`covers`)로 쓴다.

카드는 `data/cards/<과목>/<ID>.md` 에 Markdown + YAML frontmatter로 쓴다. JSON이
아닌 이유는 LaTeX 백슬래시를 이스케이프하면 사람이 못 읽기 때문이다. 형식은
[data/cards/README.md](data/cards/README.md) 참고.

폰과 패드가 같은 카드를 나눠 쓴다. **폰**은 `summary` + `formulas` + `traps` 만
(한 손, 15분, 끊겨도 되는 복습), **패드/웹**은 2단 레이아웃으로 본문 전체.
그래서 `summary` 는 요약문이 아니라 *그것만 읽어도 값이 있는 문장*이어야 한다 —
지하철에서 보는 건 이것뿐이다.

`status: draft` 인 카드는 앱에 **검수 전** 배지가 붙는다. 전기 계산은 틀리면
치명적이라 기본값이 draft다.

## 파이프라인

```bash
python3 -m venv .venv && .venv/bin/pip install -r tools/requirements.txt

.venv/bin/python tools/parse_chulje.py   # 출제기준 PDF -> data/taxonomy/*.json
.venv/bin/python tools/build_assets.py   # -> app/assets/taxonomy.json (개념 ID 부여)
.venv/bin/python tools/build_cards.py    # data/cards/*.md -> app/assets/cards.json
```

`build_cards.py` 는 taxonomy와 대조해서 **어긋나면 빌드를 멈춘다.** 카드 ID 오타,
존재하지 않는 `covers`, 과목 폴더 불일치는 오류고, 안 덮은 세세항목과 빠진 섹션은
경고다. 학습 기록이 이 ID에 매달리므로 조용히 넘어가면 안 된다.

PDF 파싱에서 데인 것들은 `parse_chulje.py` docstring에 적어뒀다 — 요약하면
(1) `pdftotext -layout` 은 한글 폭 때문에 못 쓴다, (2) 이 PDF는 45p 중 34p부터
실기 출제기준이라 잘라내야 한다, (3) 셀 안 줄바꿈의 공백은 좌표로 복원이 안 돼서
(`정전에너`+`지`는 붙이고 `전계의`+`세기`는 띄우는데 남은 폭은 오히려 붙일 쪽이
넓다) `label-fixes.json` 으로 51개를 교정한다.

## 앱

```bash
cd app
flutter run -d chrome    # 웹
flutter build web --release
```

현재 Chrome/웹만 바로 된다. iOS는 Xcode, Android는 Android SDK 설치가 필요하다.

### 학습 기능

끊어서 공부하는 직장인에 맞춰, 진척이 기기에 남는다 (`shared_preferences` —
웹은 localStorage). 상태관리 패키지 없이 `ChangeNotifier` + `ListenableBuilder`.

- **학습 완료·즐겨찾기** — 카드 화면 앱바에서 토글. 홈·과목 화면에 진척이 반영된다.
- **이어서 볼 과목** — 홈 상단이 학습이 가장 뒤처진 과목을 지목한다. 과락(과목당
  40점)이 있는 시험이라 고르게 못 봤을 때 약한 데를 짚어준다.
- **모아보기** — 즐겨찾기 / 아직 안 본 카드(이어보기)를 두 탭으로.
- **검색** — 제목·요약·항목 경로를 단어로 훑는다. 트리를 파고들 필요 없이 바로 튄다.
- **이전/다음** — 과목·검색결과·즐겨찾기 등 들어온 문맥 순서대로 카드를 넘겨본다.

**한글 폰트는 반드시 번들해야 한다.** Flutter 웹은 CanvasKit/skwasm 렌더러만 쓰고
이건 CSS 시스템 폰트를 못 써서, 폰트를 안 넣으면 한글이 전부 두부(□)로 나온다.
Pretendard Regular/Bold를 넣어뒀다 (SIL OFL 1.1).
