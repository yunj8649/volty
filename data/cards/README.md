# 이론 카드 포맷

## 단위: 세부항목 1개 = 카드 1장 (총 198장)

세세항목(680개)이 아니라 **세부항목**이 카드 단위다. 세세항목은 개별로는
두세 줄짜리 토막이라 따로 읽을 이유가 없고(`전하` 아래 `전하의 성질 / 검전기 /
쿨롱의 법칙 / 진공 중의 유전율 등`), 21개 세부항목은 세세항목이 1개뿐이라 제목
반복이다. 게다가 상당수가 **"등"으로 끝나** 공단 스스로 목록을 열어뒀다 —
세세항목은 완결된 목록이 아니라 예시다.

그래서 세세항목은 **그 카드가 반드시 덮어야 할 체크리스트**(`covers`)로 쓴다.
카드를 쓸 때 covers를 다 다뤘는지가 완성 기준이다.

## 파일 위치

```
data/cards/<과목코드>/<세부항목ID>.md      예: data/cards/em/em.1.4.md
```

과목코드: `em` 전기자기학 · `pw` 전력공학 · `mc` 전기기기 ·
`ct` 회로이론 및 제어공학 · `kec` 전기설비기술기준

## 왜 JSON이 아니라 Markdown인가

198장을 손으로 쓰고 검수해야 한다. JSON은 LaTeX 백슬래시를 전부 이스케이프해야
해서(`\\dfrac`) 사람이 못 읽는다. Markdown + YAML frontmatter로 쓰고
`tools/build_cards.py` 가 앱 에셋(JSON)으로 컴파일한다.

## 쓸 수 있는 문법

마크다운 전체가 아니다. 렌더러(`app/lib/widgets/card_body.dart`)가 직접 그리는
것만 쓴다. `flutter_markdown` 은 discontinued 라 의존하지 않는다.

| 문법 | 쓰임 |
|---|---|
| 빈 줄로 나눈 문단 | 본문 |
| `## 제목` | 섹션. 세세항목 이름과 맞으면 자동 연결(끝의 "등"은 무시) |
| `### 소제목` | 섹션 안의 소제목 |
| `$$...$$` | 블록 수식. 좁은 화면에서 가로 스크롤된다 |
| `$...$` | 인라인 수식 |
| `**굵게**` | 강조 |
| `- 항목` | 목록 |
| <code>&#124; a &#124; b &#124;</code> | 표. 첫 줄이 헤더 |

수식·굵게는 본문뿐 아니라 `summary`, `traps`, `formulas[].note`, 표 셀에서도
그대로 동작한다.

### frontmatter에서 조심할 것

**`**` 로 시작하는 줄은 따옴표로 감싸야 한다.** YAML이 `*` 를 앵커 별칭으로 읽어서
빌드가 깨진다. `tex:` 에 `*` 가 들어갈 일은 없지만 `traps:` 에서는 자주 걸린다.

```yaml
traps:
  - '**바깥 → 안 차폐는 접지가 필요 없다.** 도체로 감싸기만 하면 된다.'   # OK
  - 도체 껍질 **안쪽에** 전하가 있으면 ...                              # OK (중간은 괜찮다)
  - **바깥 → 안 차폐는** ...                                          # 깨진다
```

`tex:` 의 백슬래시(`\dfrac`)는 따옴표 없이 그냥 써도 된다. YAML 평문 스칼라는
이스케이프를 하지 않기 때문이다. 오히려 큰따옴표로 감싸면 `\d` 를 해석하려다
깨진다.

## 형식

```markdown
---
id: em.1.4                    # 세부항목 ID. taxonomy.json 과 일치해야 한다.
title: 전하
status: draft                 # draft | reviewed — 검수 전이면 앱에 '검수 전' 표시
summary: 전하는 전기 현상의 근원이고, 두 점전하 사이 힘은 거리 제곱에 반비례한다.
covers:                       # 이 카드가 덮는 세세항목. taxonomy 와 일치해야 한다.
  - em.1.4.1
  - em.1.4.2
  - em.1.4.3
  - em.1.4.4
formulas:
  - label: 쿨롱의 법칙
    tex: F = \dfrac{1}{4\pi\varepsilon_0}\dfrac{Q_1 Q_2}{r^2}
    note: 진공 중. 부호가 같으면 척력.
traps:
  - 거리에 반비례가 아니라 거리의 **제곱**에 반비례한다.
related:
  - em.1.2
---

## 전하의 성질            <!-- ## 제목이 세세항목 이름과 맞으면 자동 연결된다 -->

본문. **굵게**, $F = ma$ 인라인 수식, 블록 수식은:

$$
F = \dfrac{1}{4\pi\varepsilon_0}\dfrac{Q_1 Q_2}{r^2}
$$

## 쿨롱의 법칙

...
```

### 필드

| 필드 | 필수 | 쓰임 |
|---|---|---|
| `id` | O | taxonomy의 세부항목 ID. 학습 기록이 여기 매달린다 |
| `title` | O | 카드 제목 |
| `status` | O | `draft` 는 앱에서 '검수 전' 배지. 전기 계산은 틀리면 치명적이라 기본값 |
| `summary` | O | 한 줄. **폰 마이크로 세션과 복습에서 이것만 보인다** |
| `covers` | O | 덮는 세세항목 ID 목록. 누락되면 빌드가 경고한다 |
| `formulas` | - | 폰에서도 보이는 핵심 공식. `tex` 는 LaTeX |
| `traps` | - | 자주 틀리는 지점 |
| `related` | - | 관련 카드 ID |
| 본문 | - | `##` 섹션. **패드/웹 집중 학습에서만 펼쳐진다** |

### 폰과 패드가 나눠 갖는 것

같은 카드를 화면에 따라 다르게 쓴다.

- **폰** — `summary` + `formulas` + `traps`. 한 손으로, 15분, 끊겨도 되는 복습.
- **패드/웹** — 본문 섹션 전체. 2단 레이아웃으로 집중 학습.

그래서 `summary` 는 "카드를 요약한 문장"이 아니라 **그것만 읽어도 값이 있는 문장**
이어야 한다. 지하철에서 보는 건 이것뿐이다.
