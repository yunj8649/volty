## 전력계통의 원격조작

### 왜 필요한가

$$
\text{전국의 발전소 ・ 변전소를 \textbf{한곳에서} 보고 움직여야 한다}
$$

pw.3.4에서 본 대로 **전기는 저장이 안 되니** 실시간으로 수급을 맞춰야 한다. 그러려면
전국의 상태를 실시간으로 알고, 즉시 지령을 내려야 한다.

사람이 각 변전소에 앉아 있을 수는 없다. 그래서 **무인화 + 원격제어**다.

$$
\text{급전소} \longleftrightarrow \text{통신} \longleftrightarrow \text{전국의 변전소}
$$

### 체계

$$
\boxed{\text{현장}(\text{RTU}) \to \text{통신} \to \text{SCADA}(\text{감시・제어}) \to \text{EMS}(\text{판단・최적화})}
$$

| 계층 | 이름 | 역할 |
|---|---|---|
| **현장** | **RTU** (원격단말장치) | 측정·명령 실행 |
| **통신** | 광통신, PLC | 자료 전달 |
| **감시** | **SCADA** | 보고 움직인다 |
| **판단** | **EMS** | 무엇을 할지 정한다 |

### SCADA vs EMS — 구분하라

$$
\text{SCADA} = \text{Supervisory Control And Data Acquisition}
$$
$$
\text{EMS} = \text{Energy Management System}
$$

| | **SCADA** | **EMS** |
|---|---|---|
| 하는 일 | **감시 · 제어 · 자료 수집** | **판단 · 최적화** |
| 비유 | **눈과 손** | **머리** |
| 예 | "차단기가 열렸다", "차단기를 열어라" | "1호기 출력을 200MW로" |

$$
\text{SCADA가 \textbf{보고 움직이고}, EMS가 \textbf{무엇을 할지 정한다}}
$$

**EMS가 SCADA 위에 얹힌다.** SCADA 없이 EMS는 볼 수가 없고, EMS 없이 SCADA는
판단을 못 한다.

### EMS의 기능

pw.3.4에서 본 것들이다.

| 기능 | 내용 |
|---|---|
| **AGC** | 자동 발전 제어 |
| **AFC / LFC** | 자동 주파수 제어 |
| **EDC / ELD** | **경제 급전** (등증분연료비) |
| **상태 추정** | 측정값으로 계통 전체 상태를 추정 |
| **조류 계산** | 어디로 얼마가 흐르나 |
| **안전도 해석** | N-1 기준 만족하나 |
| 수요 예측 | |

**상태 추정**이 흥미롭다. 계통의 모든 지점을 다 잴 수는 없다. 일부 측정값과 계통
구조를 알면 **나머지를 계산으로 추정**한다. 측정 오차도 걸러낸다.

### 배전은 DAS/DMS

$$
\text{송전: EMS} \qquad / \qquad \text{배전: \textbf{DAS ・ DMS}}
$$

| 이름 | |
|---|---|
| **DAS** | 배전 자동화 시스템 |
| **DMS** | 배전 관리 시스템 |

**이름이 다르다**는 게 시험에 나온다.

배전 자동화의 핵심은 **고장 구간 자동 분리**다.

$$
\text{고장 발생} \to \text{위치 파악} \to \text{그 구간만 분리} \to \text{나머지는 우회 급전}
$$

pw.3.2의 환상·망상 배전이 이걸 가능하게 한다.

## 전력계통의 원격제어

### 텔레미터링 4요소

$$
\boxed{\text{TM} \cdot \text{TC} \cdot \text{TS} \cdot \text{TA}}
$$

| 약어 | 이름 | 무엇 |
|---|---|---|
| **TM** | Telemetering | **측정값** (전압, 전류, 전력) |
| **TC** | Telecontrol | **제어 명령** (차단기 개폐) |
| **TS** | Telesignalling | **상태** (열림/닫힘) |
| **TA** | Telealarm | **경보** (고장, 이상) |

**측정 · 제어 · 상태 · 경보** 넷이다.

**TM과 TS의 차이**가 헷갈린다.

$$
\text{TM = \textbf{아날로그}}(\text{전압 } 154.3\ \mathrm{kV}) \qquad / \qquad \text{TS = \textbf{디지털}}(\text{차단기 열림/닫힘})
$$

### 통신 수단

pw.3.4에서 본 것들이다.

| 수단 | 특징 |
|---|---|
| **광통신**(OPGW) | **주력**. 가공지선에 광섬유 |
| **PLC** | 전력선 반송. 보조 |
| 마이크로웨이브 | |
| 전용선 | |

**OPGW가 절묘하다.** 가공지선은 어차피 뇌 차폐용으로 매달아야 하는데(pw.4.1), 그
안에 광섬유를 넣으면 통신선을 따로 안 깔아도 된다.

$$
\text{가공지선}(\text{뇌 차폐}) + \text{광섬유}(\text{통신}) = \text{OPGW}
$$

### 원격제어의 위험 — 보안

$$
\text{계통 제어망이 뚫리면 \textbf{대정전}}
$$

**실제로 일어났다.** 2015년 우크라이나에서 해킹으로 변전소 차단기가 원격으로 열려
23만 가구가 정전됐다.

| 대책 | |
|---|---|
| **망 분리** | 제어망을 인터넷에서 격리 |
| 전용 프로토콜 | |
| 접근 통제·인증 | |
| **수동 조작 가능** | 최후에는 사람이 현장에서 |

**수동 조작을 남겨두는 게 중요하다.** 원격이 다 죽어도 사람이 가서 조작할 수 있어야
한다.

### 무인화의 대가

| 장점 | 단점 |
|---|---|
| **인건비 절감** | **현장 판단이 없다** |
| **신속한 대응** | 통신 두절 시 무력 |
| 전국 통합 운용 | **보안 취약** |
| 자료 축적·분석 | 설비 투자 |

**현장 판단이 없다**는 게 은근한 위험이다. 사람이 있으면 "뭔가 이상한 소리가 난다"를
알아채는데, 원격은 센서가 잡는 것만 안다.

그래서 **상태 감시**(진동, 온도, 가스, 부분방전)를 계속 늘려간다. pw.4.2의 절연
진단 기술이 원격 감시로 들어오는 것이다.
